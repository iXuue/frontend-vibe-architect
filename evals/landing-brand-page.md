# Eval: Landing Brand Page

## User Input

Design a landing page for a new AI design studio. I want it to feel memorable, premium, and future-facing. It should explain the offer, show proof, and drive people to book a call.

## Expected Routing

- Read `flows/intake.md`.
- Produce `output/requirement-summary.md`.
- Read `flows/choose-track.md`.
- Read `tracks/landing-brand.md`.
- Read `rules/reference-adaptation.md`.
- Read `rules/animation.md` if atmospheric motion or scroll effects are proposed.
- Generate design options before implementation.

## Expected Track

- Primary: `tracks/landing-brand.md`.
- Secondary: `tracks/web-product.md` only if the page includes a real interactive product demo.

## Expected Strategy

- Primary: `strategy/conversion-first.md` because the page must explain an offer and drive booking.
- Secondary: `strategy/expression-first.md` because memorability, premium tone, and future-facing brand impression are part of the requirement.
- `strategy/immersive-first.md` may be considered only if atmosphere supports the offer and does not hide proof or CTA.

## Expected Modes

- Consider `modes/dynamic-future.md` for memorability, atmosphere, and premium impression.
- Consider `modes/simple-designed.md` as a restrained premium alternative if the user wants less visual intensity.

## Expected Rules

- `rules/reference-adaptation.md`.
- `rules/examples-are-not-requirements.md` if the user provides named references later.
- `rules/animation.md` for motion and scroll behavior.
- `rules/verification.md` if implementation quality is claimed.

## Expected Output Templates

- `output/requirement-summary.md`.
- `output/design-options.md`.
- `output/design-brief.md`.
- `output/frontend-execution-plan.md` only after design direction is chosen and implementation planning is requested.

## Must Ask Before Proceeding

- What conversion action matters most if booking is not the only goal.
- What proof assets exist, such as case studies, client logos, screenshots, testimonials, or metrics.
- Whether the user wants bold atmosphere or restrained premium clarity if the direction is ambiguous.

## Must Not Do

- Do not build an app dashboard instead of a landing page.
- Do not use generic decoration without connecting it to the brand offer.
- Do not copy visual references if the user later names websites or products.

## Passing Output Should Include

- A landing-brand track choice.
- Options that differ in narrative, layout, and motion approach.
- A clear CTA structure.
- Risks around asset availability and over-atmospheric design.
