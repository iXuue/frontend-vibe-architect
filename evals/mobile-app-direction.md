# Eval: Mobile App Direction

## User Input

I need a mobile app interface for a habit and focus product. It should have daily tasks, focus sessions, progress, reminders, and a calm but designed feeling.

## Expected Routing

- Read `flows/intake.md`.
- Produce `output/requirement-summary.md`.
- Read `flows/choose-track.md`.
- Read `tracks/mobile-direction.md`.
- Read `rules/reference-adaptation.md`.
- Read `rules/animation.md` if habit streaks, session transitions, or progress motion are proposed.

## Expected Track

- Primary: `tracks/mobile-direction.md`.
- Secondary: `tracks/data-visualization.md` may be considered for progress and habit statistics.

## Expected Strategy

- Primary: `strategy/efficiency-first.md` because the product supports repeated daily tasks, focus sessions, reminders, and progress checks.
- Secondary: `strategy/trust-first.md` may be considered because calm focus products need reliable, low-friction, confidence-building behavior.
- `strategy/data-insight-first.md` may be considered only for progress and habit analytics, not as the whole product strategy.

## Expected Modes

- Prefer `modes/simple-designed.md` for calm repeated use.
- Consider limited `modes/dynamic-future.md` only for progress feedback, focus session ambience, or completion moments.

## Expected Rules

- `rules/reference-adaptation.md`.
- `rules/animation.md` if motion is included.
- `rules/verification.md` if implementation or platform quality is claimed.

## Expected Output Templates

- `output/requirement-summary.md`.
- `output/design-options.md` if the visual direction is not yet chosen.
- `output/design-brief.md` after direction selection.
- `output/frontend-execution-plan.md` if the user asks to build it.

## Must Ask Before Proceeding

- Whether the target is native mobile app, mobile web, or mini program direction.
- Which first-version screens are required.
- Whether reminders and notifications are in scope for UI only or product behavior too.

## Must Not Do

- Do not claim native iOS, Android, or mini program compliance without verification.
- Do not make the interface visually busy if the stated goal is calm focus.
- Do not skip mobile constraints such as touch targets and narrow viewport behavior.

## Passing Output Should Include

- A mobile-direction track choice.
- Clear screen list and core flow.
- A calm visual mode recommendation with controlled motion.
- Open questions about target platform and first-version scope.
