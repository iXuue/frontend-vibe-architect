# Example: Informal Visual Example

## User Request

I want the interface to feel premium, airy, layered, and future-facing. It can be similar to some modern system UI feelings, but do not copy anything. I do not know the professional design terms.

## Expected Routing

Read:

- `flows/intake.md`
- `output/requirement-summary.md`
- `flows/choose-track.md`
- `rules/examples-are-not-requirements.md`
- `rules/reference-adaptation.md`
- `modes/dynamic-future.md`
- `modes/simple-designed.md`
- `flows/design-options.md`
- `output/design-options.md`

Track routing depends on product context:

- If the product is a tool, read `tracks/web-product.md`.
- If the product is a website or launch page, read `tracks/landing-brand.md`.
- If the platform is mobile, read `tracks/mobile-direction.md`.
- If the product is data-heavy, read `tracks/data-visualization.md`.

## Expected Requirement Interpretation

- Hard requirements:
  - Not enough information yet; product type and platform must be identified.
- Reference examples:
  - "Modern system UI feelings" is a reference example, not a hard requirement.
- Visual intent:
  - Premium.
  - Airy.
  - Layered.
  - Future-facing.
- Forbidden directions:
  - Do not copy a named system, product, or trend.
  - Do not turn any single material or visual effect into a mandatory rule.
- Design variables:
  - Hierarchy.
  - Depth.
  - Contrast.
  - Motion intensity.
  - Information density.
  - Readability.
  - Atmosphere.
  - Platform fit.
- Open questions:
  - What product is this for?
  - Is this a website, web app, mobile app, mini program, desktop app, or dashboard?
  - Which matters more: visual memorability or repeated-use efficiency?
  - Are any example references mandatory, or only mood references?

## Expected Design Direction Shape

Do not generate a single fixed style from the example.

Generate options such as:

- Layered Premium Product: simple designed structure with controlled depth and strong hierarchy.
- Dynamic Future Brand Moment: expressive atmosphere and motion for first impression.
- Quiet System-Like Workspace: restrained layout, strong typography, precise spacing, minimal motion.

Each option should explain:

- Which visual variables it uses.
- Which track it fits.
- Which mode it uses.
- What it deliberately avoids.
- What needs user confirmation.

## Expected Output Shape

First output should prioritize:

- Requirement summary.
- Separation of hard requirements and reference examples.
- Clarifying questions if product/platform is unclear.
- 2-4 directions only after enough context exists.

If context is insufficient, ask concise questions before producing a full design plan.

## Notes On What Not To Do

- Do not say the user asked for a specific named style.
- Do not create a dedicated rule for the example.
- Do not reduce "premium, airy, layered, future-facing" to one material treatment.
- Do not ignore product type.
- Do not skip clarification when product and platform are missing.
