# Layout Language

## Use When

Use when the user describes structure, density, spacing, information hierarchy, simplicity, richness, focus, or scan speed.

## What User Might Say

- Make it simple.
- Don't make it empty.
- It should be easy to scan.
- I want it rich but not messy.
- Make it feel spacious.
- It needs to show a lot of information.

## What It May Mean

- Adjust information density.
- Clarify hierarchy.
- Group related content.
- Use predictable navigation.
- Reduce competing sections.
- Choose a layout pattern that matches workflow complexity.
- Balance whitespace with useful content.

## What It Does Not Necessarily Mean

- Simple does not mean fewer required workflows.
- Spacious does not mean oversized hero sections.
- Rich does not mean cluttered.
- Dense does not mean unreadable.
- Minimal does not mean incomplete.

## Clarifying Questions

- What is the primary task users must complete?
- How much information must be visible at once?
- Is speed of scanning more important than visual storytelling?
- Which screens or sections are required in the first version?

## Design Variables To Adjust

- Grid structure.
- Sidebar, topbar, tabs, or stepper navigation.
- Section order.
- Card and panel usage.
- Table, list, chart, or canvas density.
- Breakpoints and stacking behavior.
- Empty, loading, and error state placement.

## Suitable Tracks

- `tracks/web-product.md` for app and dashboard structure.
- `tracks/data-visualization.md` for dense metrics and scanning.
- `tracks/landing-brand.md` for narrative section flow.
- `tracks/mobile-direction.md` for focused sequential flows.
- `tracks/desktop-direction.md` for multi-pane professional tools.

## Suitable Modes

- `modes/simple-designed.md` when clarity, density, and repeated use matter.
- `modes/dynamic-future.md` only when layout can support atmosphere without hurting task flow.

## Risks

- Using decorative cards for whole page structure.
- Making operational tools look like landing pages.
- Hiding core workflows behind visual storytelling.
- Letting long labels or dynamic content break layout.

## Output Guidance

- Specify page or screen structure before visual polish.
- State the information hierarchy.
- Identify density expectations.
- Include responsive behavior and overflow rules.
