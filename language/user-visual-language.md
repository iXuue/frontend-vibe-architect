# User Visual Language

## Use When

Use when the user uses broad visual words such as premium, clean, high-end, modern, designed, playful, warm, calm, sharp, bold, elegant, simple, cool, lively, or professional.

## What User Might Say

- Make it look premium.
- It should be simple but not plain.
- I want it clean and professional.
- Make it feel warm and trustworthy.
- I want something bold and memorable.
- It should have design, not just default components.

## What It May Mean

- Stronger visual hierarchy.
- More deliberate spacing and alignment.
- Fewer competing colors.
- Better typography roles.
- Clearer component states.
- More controlled density.
- More polished icon, image, material, or motion treatment.
- A product tone that matches the audience and task.

## What It Does Not Necessarily Mean

- Dark mode.
- Glass effects.
- Gradients.
- Large empty space.
- Heavy animation.
- Decorative visuals.
- Fewer features.
- Copying a named reference product.

## Clarifying Questions

- What should users feel first: confidence, speed, warmth, authority, delight, focus, or curiosity?
- Is the product used once, occasionally, or every day?
- Should the interface feel quiet and utilitarian, expressive and memorable, or somewhere between?
- Which matters more: visual impact or task speed?

## Design Variables To Adjust

- Typography scale and weight.
- Spacing rhythm.
- Color roles and contrast.
- Surface depth and borders.
- Component density.
- Image or icon treatment.
- Motion level.
- Empty state and feedback language.

## Suitable Tracks

- All tracks.
- Interpret the same word differently by track. A premium landing page can be expressive; a premium admin tool should usually be restrained and precise.

## Suitable Modes

- `modes/simple-designed.md` for clarity, restraint, and repeated use.
- `modes/dynamic-future.md` only when visual impact, atmosphere, or advanced interaction is truly useful.

## Risks

- Treating taste words as hard requirements.
- Over-styling an operational product.
- Making a simple request visually empty.
- Choosing visual effects before understanding product context.

## Output Guidance

- Restate the user's language as visual intent, not confirmed implementation.
- Explain the design variables being adjusted.
- Offer options when the same word can support multiple valid directions.
- Mark unclear taste terms as assumptions or open questions.
