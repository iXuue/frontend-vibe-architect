# Mode: Dynamic Future

## Use When

Use when expression, brand memory, spatial storytelling, advanced interaction, or dynamic data atmosphere improves the product.

This mode is appropriate when the interface benefits from being memorable, immersive, responsive, or visually directional.

## Design Intent

Create a future-facing interface that feels alive, precise, layered, and product-aware.

The goal is not to apply a fixed trend. The goal is to use depth, motion, responsiveness, and visual atmosphere to help users understand what the product is, what is changing, and where to act.

## Visual Variables

Tune these variables instead of copying a style example:

- Spatial depth: flat, layered, dimensional, immersive.
- Surface treatment: quiet, luminous, tactile, technical, editorial.
- Motion intensity: static, subtle, guided, cinematic, reactive.
- Background activity: none, ambient, data-driven, generative.
- Contrast level: restrained, sharp, high-contrast, display-led.
- Information density: sparse, balanced, dense, monitoring-grade.
- Signature element: one memorable object, interaction, layout move, or dynamic system.

## Motion Level

Use motion when it clarifies:

- State change.
- Cause and effect.
- Spatial continuity.
- Data flow.
- Hierarchy.
- Progress.
- User feedback.

Allowed motion levels:

- Low: hover, focus, state feedback, small transitions.
- Medium: coordinated entrances, panel transitions, guided reveal, lightweight ambient motion.
- High: scroll storytelling, data-driven atmosphere, reactive backgrounds, timeline-based sequences.

High motion requires explicit performance and reduced-motion safeguards.

## Suitable Tracks

Works best with:

- `tracks/landing-brand.md` when the page needs a strong first impression.
- `tracks/data-visualization.md` when motion clarifies live state, flow, or anomaly.
- `tracks/web-product.md` when the product itself is creative, AI-driven, exploratory, or brand-sensitive.

Use carefully with:

- `tracks/mobile-direction.md`, because small screens and performance limits reduce motion budget.
- `tracks/desktop-direction.md`, because professional tools usually need stable density and predictable controls.

## Not Suitable For

Avoid this mode when:

- Users need long-session operational efficiency.
- The interface is dense with tables, forms, settings, or compliance workflows.
- Motion would delay task completion.
- The visual atmosphere would reduce readability.
- The product goal is trust through restraint rather than memorability.
- The user only gave a visual example and did not confirm it as a hard direction.

## Required Safeguards

- Provide reduced-motion fallback.
- Provide mobile performance fallback.
- Keep important content visible without waiting for animation.
- Keep text readable over all backgrounds and dynamic layers.
- Avoid motion that blocks reading, form completion, or navigation.
- Verify contrast and overlap in desktop and mobile viewports.
- Keep one dominant visual idea; do not stack multiple spectacle effects.

## Do

- Tie every effect to product meaning.
- Use motion to express hierarchy, state, causality, data, or spatial continuity.
- Choose a signature visual move that fits the product.
- State the implementation difficulty and performance risk.
- Explain why this mode serves the requirement better than a quieter mode.

## Do Not

- Do not default to dark tech UI.
- Do not use a named product, operating system, material, website, or trend as a hard rule unless the user confirms it.
- Do not use decorative animation without a job.
- Do not hide core content behind delayed reveals.
- Do not make the whole interface atmospheric if the user needs a work surface.
- Do not let visual ambition override accessibility, responsiveness, or maintainability.

## Output Notes

When this mode is selected, design options and execution plans must mention:

- Motion level.
- Signature dynamic element.
- Background or atmosphere strategy.
- Reduced-motion fallback.
- Mobile performance fallback.
- Readability safeguards.
- Implementation risk.
- Why the dynamic treatment fits the product goal.
