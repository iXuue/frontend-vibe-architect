# Mode: Simple Designed

## Use When

Use when speed, clarity, repetition, density, accessibility, or operational confidence matters more than visual spectacle.

This mode is appropriate when users need to understand, compare, decide, operate, edit, monitor, or repeat tasks without visual fatigue.

## Design Intent

Create a quiet but clearly designed interface.

The goal is not to make the UI plain. The goal is to remove noise while preserving hierarchy, rhythm, typography, state clarity, and one memorable design choice.

## Visual Variables

Tune these variables deliberately:

- Typographic hierarchy: utilitarian, editorial, technical, warm, formal.
- Density: relaxed, balanced, compact, high-density.
- Surface contrast: low, medium, strong.
- Color use: monochrome, restrained accent, structured palette, status-led palette.
- Shape language: sharp, modest radius, soft, system-like.
- Dividers and grouping: spacing-led, line-led, panel-led, table-led.
- Signature element: one memorable but quiet detail in layout, type, color, iconography, or interaction.

## Motion Level

Use low or medium-low motion.

Appropriate motion:

- Hover and focus feedback.
- State transitions.
- Simple panel changes.
- Loading and progress feedback.
- Small confirmation or error feedback.

Avoid motion that creates spectacle. Motion should reduce uncertainty, not create atmosphere.

## Suitable Tracks

Works best with:

- `tracks/web-product.md` for SaaS, dashboards, admin panels, AI tools, and creative workspaces.
- `tracks/desktop-direction.md` for professional tools and dense workspaces.
- `tracks/data-visualization.md` when chart readability and comparison matter.
- `tracks/mobile-direction.md` when small-screen clarity and short flows matter.

Use selectively with:

- `tracks/landing-brand.md`, especially for premium, editorial, technical, or trust-led brand pages.

## Not Suitable For

Avoid this mode as the only direction when:

- The user explicitly needs a strong launch moment or highly memorable brand impression.
- The product's selling point is immersive interaction or visual experimentation.
- The interface would become too generic without a signature element.
- The user asked for multiple expressive directions and no quiet option is useful.

## Required Safeguards

- Include a clear hierarchy, not just fewer elements.
- Maintain readable contrast.
- Preserve enough density for the product type.
- Define states for controls, forms, errors, empty screens, and loading where relevant.
- Include one memorable visual choice so the result does not feel unfinished.
- Verify mobile text wrapping and control sizes.

## Do

- Use typography, spacing, alignment, and state design as primary design materials.
- Make the main task obvious.
- Keep visual decisions consistent across components.
- Use restrained color with clear roles.
- Explain the one design choice that keeps the interface distinctive.
- Prefer stability and scanability for repeated-use tools.

## Do Not

- Do not treat simple as low-effort.
- Do not remove necessary structure for minimalism.
- Do not make everything gray, flat, and anonymous.
- Do not omit empty, loading, error, and success states.
- Do not use tiny type, weak contrast, or excessive whitespace to signal elegance.
- Do not ignore brand or product personality.

## Output Notes

When this mode is selected, design options and execution plans must mention:

- Information hierarchy strategy.
- Typography role.
- Density level.
- Color role.
- Component and state strategy.
- The single memorable visual choice.
- Why a quiet mode fits the product goal.
