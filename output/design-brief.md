# Output: Design Brief

## Use When

Use after the user confirms a design direction, or after the agent makes a low-risk recommendation and explains why user confirmation is not required.

## Required Shape

```markdown
## Design Brief

- Chosen direction:
- Product rationale:
- Target platform:
- Primary track:
- Secondary track:
- Visual mode:
- Information architecture:
- Component language:
- Design tokens:
- Motion rules:
- Responsive rules:
- Accessibility notes:
- Risks:
- User confirmations:
- Assumptions:
```

## Do Not Omit

- Do not omit user confirmations for broad work.
- Do not omit assumptions.
- Do not include visual choices that cannot be traced to the requirement, selected option, or user confirmation.
- Do not claim platform-native compliance unless specifically verified.
