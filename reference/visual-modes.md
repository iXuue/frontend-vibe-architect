# Visual Modes

## Dynamic Future Mode

Use for expressive, cinematic, high-motion, future-facing interfaces.

Allowed tools:

- Optical depth.
- Generative or data-driven background layers.
- GSAP timelines.
- ScrollTrigger for storytelling pages.
- Algorithmic backgrounds.
- High-quality hover, focus, and state transitions.

Best fit:

- AI products where atmosphere supports trust or capability.
- Creative tools where dynamic visuals reinforce creation.
- Brand or launch pages where first impression matters.
- Data visualization where motion can clarify change or flow.

Constraints:

- Use motion for meaning.
- Keep text readable.
- Add reduced-motion fallback.
- Add mobile performance fallback.
- Do not hide core content until animation completes.
- Do not use visual spectacle to compensate for unclear product structure.

## Simple Designed Mode

Use for interfaces that should be quiet, efficient, and still designed.

Traits:

- Strong typography.
- Clean information hierarchy.
- Restrained color.
- Few animations.
- One memorable visual element.
- Precise spacing.
- Stable layout.

Best fit:

- Dashboards.
- Admin panels.
- Long-use SaaS tools.
- Productivity products.
- Data-heavy workflows.
- Review, approval, monitoring, and editing interfaces.

This is not a rough version. It is a deliberate low-noise design mode.

## Choosing A Mode

Choose Dynamic Future Mode when expression, brand memory, spatial storytelling, or dynamic data atmosphere improves the product.

Choose Simple Designed Mode when speed, clarity, repetition, density, accessibility, or operational confidence matters more than visual spectacle.

When uncertain, offer both modes as options and state the tradeoff.
