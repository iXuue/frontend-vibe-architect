# Strategy: Conversion First

## Use When

Use when the interface must persuade users to act, understand an offer, sign up, book, buy, subscribe, or start a trial.

## Product Fit

- Landing pages.
- Campaign pages.
- Product launch pages.
- Pricing pages.
- Lead generation flows.
- Onboarding entry points.

## Poor Fit

- Dense admin dashboards.
- Long-session professional tools.
- Interfaces where repeated workflow speed matters more than first impression.
- Data analysis surfaces.

## Design Priority

- Offer clarity.
- Message hierarchy.
- Trust proof.
- Action path.
- Objection handling.
- Momentum.

## Layout Tendency

- Strong first viewport.
- Clear section narrative.
- Visible primary CTA.
- Proof, benefits, use cases, and objections ordered by decision flow.
- Avoid app-like density unless demo is central.

## Visual Tendency

- More expressive than operational tools.
- Strong contrast around CTA and proof.
- Product or offer should be visually concrete.
- Brand personality can be more pronounced.

## Motion Tendency

- Motion can guide attention and create momentum.
- Scroll motion may be useful when it supports narrative.
- Avoid motion that delays understanding or CTA access.

## Interaction Tendency

- Clear CTA states.
- Low-friction forms.
- Pricing or package comparison when relevant.
- Product demo interactions only if they support conversion.

## Suitable Tracks

- `tracks/landing-brand.md`
- `tracks/web-product.md` as secondary if product demo matters.
- `tracks/mobile-direction.md` for app install or onboarding conversion.

## Suitable Modes

- `modes/dynamic-future.md` when memorability and atmosphere help.
- `modes/simple-designed.md` when trust, clarity, or premium restraint matters more.

## Language Interpretation

- "High-end" should support credibility and desirability.
- "Bold" should strengthen the offer, not distract from it.
- "Simple" should make the decision easier, not remove necessary proof.

## Risks

- Visual spectacle hiding the offer.
- Weak proof.
- Too many CTAs.
- Generic hero sections.
- Confusing brand mood with conversion clarity.

## Output Guidance

- Explain the conversion path.
- Include CTA hierarchy and proof strategy.
- State what assets are needed.
- Treat missing proof as a risk.
