# Strategy: Data Insight First

## Use When

Use when the interface must help users monitor, analyze, compare, diagnose, report, or make decisions from data.

## Product Fit

- Dashboards.
- Analytics tools.
- Monitoring systems.
- Financial or operational reporting.
- Data products.
- Large-screen displays.

## Poor Fit

- Simple brand pages.
- Pure storytelling pages with little real data.
- Interfaces where emotion matters more than accuracy.
- Decorative experiences that do not need analysis.

## Design Priority

- Sensemaking.
- Accuracy.
- Scan speed.
- Comparison.
- Context.
- Alerting and prioritization.

## Layout Tendency

- Clear data hierarchy.
- Stable grids.
- Tables, charts, filters, summaries, drilldowns, and annotations.
- Avoid card-heavy decoration that weakens comparison.

## Visual Tendency

- Data-first contrast.
- Color has semantic meaning.
- Reduce non-data ink.
- Typography supports numbers and labels.
- Avoid decorative palettes that confuse status.

## Motion Tendency

- Use motion for data changes, drilldowns, loading, and transitions between views.
- Avoid cinematic or idle motion.
- Keep changes understandable and non-distracting.

## Interaction Tendency

- Filtering, sorting, drilldown, comparison, annotations, export, and alerts.
- Clear selected states.
- Preserve context during navigation.

## Suitable Tracks

- `tracks/data-visualization.md`
- `tracks/web-product.md`
- `tracks/desktop-direction.md`

## Suitable Modes

- Usually `modes/simple-designed.md`.
- `modes/dynamic-future.md` only for controlled data motion, spatial monitoring, or high-level status display.

## Language Interpretation

- "Dynamic" should usually mean live data and understandable updates.
- "Beautiful" should not reduce readability.
- "Futuristic" should not weaken chart clarity.

## Risks

- Visual effects competing with data.
- Color misuse.
- Poor responsive behavior for charts and tables.
- Missing loading, empty, or error states.
- Claiming insight without showing data relationships.

## Output Guidance

- Define data hierarchy and decision path.
- Specify chart/table roles.
- Include states, filters, and responsive constraints.
- Require verification for readability and layout stability.
