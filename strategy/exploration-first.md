# Strategy: Exploration First

## Use When

Use when users need to browse, compare, discover, filter, learn, or navigate many possible paths.

## Product Fit

- Marketplaces.
- Content libraries.
- Documentation portals.
- Product catalogs.
- Learning platforms.
- Creative galleries.
- Search and discovery tools.

## Poor Fit

- Single-goal conversion pages.
- High-speed internal tools.
- Linear transaction flows.
- Interfaces where too many choices create risk.

## Design Priority

- Discovery.
- Orientation.
- Comparison.
- Progressive disclosure.
- Clear filtering and wayfinding.
- Encouraging useful paths without overwhelming users.

## Layout Tendency

- Browseable grids, lists, filters, categories, tabs, search, related items, and saved views.
- Clear item previews.
- Good empty and no-results states.
- Avoid forcing one linear path when exploration is needed.

## Visual Tendency

- Content-led hierarchy.
- Clear metadata.
- Balanced density.
- Visual variety may help, but should not reduce comparison.

## Motion Tendency

- Use motion for filtering, reveal, preview, and continuity.
- Avoid motion that makes comparison difficult.
- Keep transitions fast and reversible.

## Interaction Tendency

- Search, filter, sort, save, compare, preview, and backtracking.
- Maintain user orientation.
- Preserve state after navigation.

## Suitable Tracks

- `tracks/web-product.md`
- `tracks/mobile-direction.md`
- `tracks/landing-brand.md` for content-rich brand sites.
- `tracks/data-visualization.md` when exploration is analytical.

## Suitable Modes

- `modes/simple-designed.md` for clear browsing and comparison.
- `modes/dynamic-future.md` when discovery needs a more memorable or spatial experience.

## Language Interpretation

- "Rich" should mean useful paths and content depth, not clutter.
- "Fun" should not reduce findability.
- "Modern" can mean better filtering and preview patterns, not just visuals.

## Risks

- Too many options with weak hierarchy.
- Decorative layouts that reduce comparison.
- Losing filter state.
- Making users restart exploration.

## Output Guidance

- Define the browse model.
- Include filtering, search, preview, and empty-state behavior.
- Explain how users stay oriented.
