# Strategy: Immersive First

## Use When

Use when the interface should create atmosphere, narrative, spatial feeling, presence, or high-impact presentation, and when this supports the product goal.

## Product Fit

- Launch pages.
- Experiential brand pages.
- Creative showcases.
- Interactive stories.
- Product demos.
- Campaigns.
- Certain education, entertainment, or culture experiences.

## Poor Fit

- Dense dashboards.
- High-frequency work tools.
- Risk-sensitive products.
- Forms, settings, admin systems, and transaction-heavy flows.
- Interfaces where users need fast scanning.

## Design Priority

- Atmosphere.
- Narrative.
- Presence.
- Memorable moments.
- Spatial hierarchy.
- Emotional pacing.

## Layout Tendency

- Large visual moments.
- Section-based narrative.
- Full-bleed media or interactive scenes when justified.
- Clear escape routes to content and action.
- Avoid trapping users in effects.

## Visual Tendency

- Strong media or scene direction.
- Depth, scale, contrast, and atmospheric composition.
- Brand or product should remain visible and concrete.
- Avoid generic abstract backgrounds.

## Motion Tendency

- Motion can be central.
- Use scroll, reveal, parallax, 3D, or scene transitions only when they support story or product understanding.
- Provide reduced-motion fallback.

## Interaction Tendency

- Exploratory but understandable.
- Keep primary action accessible.
- Avoid blocking content behind complex interaction.
- Preserve orientation.

## Suitable Tracks

- `tracks/landing-brand.md`
- `tracks/web-product.md` only for product demos or creative tools.
- `tracks/mobile-direction.md` only for consumer experiences where immersion is the value.

## Suitable Modes

- Often `modes/dynamic-future.md`.
- Pair with `modes/simple-designed.md` when content sections need restraint and readability.

## Language Interpretation

- "Immersive" should mean product-relevant atmosphere, not arbitrary effects.
- "Dynamic" can mean narrative pacing, not constant movement.
- "Premium" still requires clarity, not only visual spectacle.

## Risks

- Slow performance.
- Poor accessibility.
- Weak content clarity.
- Motion fatigue.
- Generic atmospheric visuals.
- Implementation cost.

## Output Guidance

- Explain why immersion supports the requirement.
- Define where immersion starts and stops.
- Include fallback and verification requirements.
- State implementation difficulty honestly.
