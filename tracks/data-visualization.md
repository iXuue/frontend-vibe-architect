# Track: Data Visualization

## Use When

Use for analytics dashboards, monitoring displays, metric pages, operational status views, reporting tools, and large-screen data displays.

Choose this track when the user's main job is to understand data, detect change, compare values, monitor state, or act on anomalies.

## Primary Goal

Make data understandable, comparable, and actionable.

The design should clarify hierarchy, time, state, and anomaly before adding atmosphere.

## Design Priorities

- Glanceable hierarchy.
- Clear metric definitions.
- Live or time-based state clarity.
- Strong contrast.
- Chart readability.
- Alert and anomaly states.
- Filtering and time range controls.
- Motion only when it clarifies change, flow, or causality.

## Layout Defaults

- Put the most important status, KPI, or anomaly near the top.
- Group related metrics and charts by decision, not by decoration.
- Use overview-first, then drill-down.
- Keep legends, labels, axes, and units readable.
- For large screens, design for distance and scanning.
- For dashboards, avoid overloading a single viewport with unrelated charts.

## Interaction Defaults

- Provide time range controls, filters, comparison states, and drill-down when relevant.
- Use hover or focus details for secondary information.
- Keep alerts and anomalies actionable.
- Allow users to distinguish normal, warning, critical, stale, and loading states.
- Ensure charts remain understandable without color alone.

## Visual Risk

- Decorative data effects that reduce legibility.
- Futuristic visuals that obscure exact values.
- Too many charts without a decision hierarchy.
- Color palettes that make categories or severity unclear.
- Motion that implies change where the data does not support it.

## Do

- Identify what decision the data supports.
- Define the primary metric or status.
- Distinguish real-time, historical, forecast, and static data.
- Include empty, loading, stale, and error states.
- Use motion only to clarify data meaning.

## Do Not

- Do not use chart decoration that makes values harder to read.
- Do not rely only on color to show severity.
- Do not hide units, labels, axes, or time range.
- Do not add animated backgrounds behind dense charts.
- Do not treat a monitoring screen like a brand landing page.

## Output Notes

When this track is selected, design options and execution plans must mention:

- Primary decision or monitoring goal.
- Main metric hierarchy.
- Chart or visualization types.
- State and severity model.
- Filter and time controls.
- Accessibility and contrast risks.
- Verification for labels, overflow, and dynamic state.
