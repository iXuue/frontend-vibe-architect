# Track: Landing Brand

## Use When

Use for websites, landing pages, launch pages, product storytelling, campaign pages, portfolios, and brand-led first impressions.

Choose this track when the user's main goal is attention, understanding, trust, positioning, conversion, or narrative.

## Primary Goal

Make the product, offer, person, or brand legible and memorable quickly.

The first viewport should communicate what this is and why it matters, while hinting at the next section.

## Design Priorities

- Strong first-viewport signal.
- Clear value proposition.
- Distinct visual identity.
- Narrative section flow.
- Credible proof, examples, or product evidence.
- Mobile readability.
- Clear calls to action.
- One memorable signature element tied to the subject.

## Layout Defaults

- Start with an expressive first viewport, not a generic split hero by default.
- Structure the page as a sequence: intro, proof, product detail, use case, trust, action.
- Use full-width sections or strong editorial rhythm instead of stacked decorative cards.
- Let imagery, product UI, live demos, or interactive moments carry the subject when available.
- Keep the next section partially discoverable where possible.

## Interaction Defaults

- Use scroll, hover, reveal, and transition effects only when they clarify the story.
- Keep calls to action visible and predictable.
- Provide smooth section transitions for long pages.
- Make forms and conversion actions simple.
- Use reference images when visual ambiguity is high.

## Visual Risk

- Looking like a generic SaaS template.
- Using big gradients, floating cards, and abstract decoration without product evidence.
- Making a beautiful page that does not explain the offer.
- Overusing scroll effects so reading becomes harder.
- Treating a user-provided example as a required visual style instead of a clue.

## Do

- Identify the subject's actual world: product, market, audience, materials, vocabulary, and proof.
- Make one distinctive design choice that fits the subject.
- Show the product, result, or offer early.
- Match visual intensity to the brand's credibility needs.
- Include mobile-first section logic.

## Do Not

- Do not default to a generic hero plus feature-card layout.
- Do not hide the actual product behind atmospheric visuals.
- Do not use numbers, badges, or decorative labels unless they encode real information.
- Do not add motion that competes with the message.
- Do not use a style trend as a substitute for positioning.

## Output Notes

When this track is selected, design options and execution plans must mention:

- First-viewport concept.
- Signature visual element.
- Section sequence.
- CTA hierarchy.
- Proof or trust elements.
- Mobile section behavior.
- Any reference-image needs.
